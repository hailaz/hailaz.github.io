## charging-sim 到达间隔→充电间隔 模型修正

**问题**：原来的 `arrivalInterval`（到达间隔）控制的是车辆相继到达站点的时间间隔，但实际应该是**充电间隔**——即前车拔枪离开 + 后车停车插枪的非充电时间。

**模型变更**：
- 旧模型：车辆按固定间隔到达站点，有空枪就充（`arrivalTime` 预计算）
- 新模型：所有车辆随时可用排队，枪位空出后有**冷却期**（`cooldownUntil`），冷却期过后下一辆车才能开始充电

**改动**：
- **engine.js**：`arrivalInterval` → `chargingInterval`，注释更新说明这是前车拔枪离开+后车停车插枪的时间
- **app.js**：
  - 移除车辆 `arrivalTime` 预计算，所有车辆随时可充
  - 枪位新增 `cooldownUntil` 字段（初始为 0）
  - `tick()` 排队→充电：检查 `g.cooldownUntil <= store.simTime`
  - 车辆充完后设置 `gun.cooldownUntil = simTime + chargingInterval * 60`
  - 结束条件从 `allArrived` 改为 `noQueuing`
  - 结论文本中 `arrivalInterval` → `chargingInterval`
- **index.html**：UI 标签"到达间隔"→"充电间隔"，提示改为"前车拔枪 → 后车插枪"，排队区移除到达时间显示
